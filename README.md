# Rabbit Hole Theater / Toto v0.22

SillyTavern 第三方扩展：每轮在回复末尾生成由 `<!-- TOTO_START -->` / `<!-- TOTO_END -->` 包裹的兔子洞小剧场。

## 生成模式

- **跟随主模型生成**：当前默认模式。主模型同时生成正文和兔子洞，最贴合主模型上下文、预设和世界书。
- **使用副 API 独立生成**：主模型只生成正文，插件再把本轮正文、最近聊天、抽取结果、冷却摘要和兔子洞规则发送给副 API，由副 API 单独生成兔子洞并追加到当前回复后面。

副 API 支持尽量兼容：OpenAI 兼容、Gemini 兼容、Claude / Anthropic 兼容、自定义。模型列表能拉则拉，拉取失败时手动填写模型名即可。

副 API 默认设置：

- 温度：0.95
- 最大输出 token：16000
- 参考上下文：本轮正文 + 最近5轮聊天

## 不发送小剧场正则

查找正则表达式：

```regex
/<!--\s*TOTO_START\s*-->[\s\S]*?<!--\s*TOTO_END\s*-->\s*/gi
```

推荐设置：替换留空／勾选 AI输出／勾选 仅格式提示词。

这样聊天界面仍可显示兔子洞，但后续发给 API 的上下文会过滤 `<!-- TOTO_START -->...<!-- TOTO_END -->`，减少小剧场 HTML 堆积。
